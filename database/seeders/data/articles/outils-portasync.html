PortaSync — Service de synchronisation PNM

PortaSync est le service applicatif qui assure la synchronisation entre la base de données PortaDB et les systèmes inter-opérateurs. C'est le composant qui gère concrètement les échanges de fichiers PNMDATA avec les autres opérateurs du GPMAG.

Rôle principal

PortaSync est le pont opérationnel entre le monde interne (PortaDB, ESB) et le monde externe (sFTP inter-opérateurs) :

  • Export : extraction des tickets depuis PortaDB, génération des fichiers PNMDATA au format normalisé, dépôt sur les serveurs sFTP des opérateurs partenaires
  • Import : collecte des fichiers PNMDATA reçus des autres opérateurs, parsing, validation, et injection des tickets dans PortaDB
  • Synchronisation : gestion du processus PNMSYNC hebdomadaire (extraction, comparaison, détection de divergences)
  • ACR/ERR : génération des accusés de réception et fichiers d'erreur pour chaque fichier reçu

Hébergement

PortaSync est hébergé sur le serveur vmqproportasync01, distinct du serveur PortaDB pour des raisons de sécurité et de performance :

  • Serveur dédié aux échanges réseau (sFTP) et au traitement des fichiers
  • Accès réseau ouvert vers les serveurs sFTP des opérateurs partenaires
  • Connexion sécurisée vers PortaDB (vmqproportawebdb01) via le réseau interne

Connexion au BTCTF

PortaSync interagit avec le BTCTF (Bureau Technique Central des Télécommunications Françaises), la plateforme nationale de référence pour la portabilité :

  • Transmission périodique des données de portage de la zone Antilles-Guyane vers le référentiel national
  • Récupération des mises à jour nationales impactant les numéros portés inter-zones (cas d'un numéro métropolitain porté en outre-mer)
  • Synchronisation des tables de routage nationales

Scripts principaux

PortaSync orchestre plusieurs scripts exécutés selon le calendrier des vacations :

  • portasync-export.sh : génère les fichiers PNMDATA à partir des tickets en attente dans PortaDB
  • portasync-push.sh : dépose les fichiers sur les serveurs sFTP distants (authentification par clé SSH)
  • portasync-pull.sh : collecte les fichiers déposés par les opérateurs sur notre serveur sFTP
  • portasync-import.sh : parse et importe les tickets reçus dans PortaDB
  • portasync-acr.sh : génère et envoie les accusés de réception
  • portasync-sunday.sh : script spécifique pour la synchronisation PNMSYNC du dimanche

Logs et diagnostic

PortaSync produit des logs détaillés pour chaque opération, stockés dans /var/log/portasync/. Ces logs incluent : les fichiers envoyés/reçus, le nombre de tickets traités, les erreurs rencontrées, et les temps de traitement. Ils constituent la première source d'information en cas d'incident sur les échanges inter-opérateurs.
