WHERE — Filtrer les résultats

Pourquoi filtrer ?

Sans filtre, SELECT * FROM porta_portage retourne tous les portages. En investigation PNM, on cherche des cas précis : un numéro particulier, un opérateur, une période. C'est le rôle de WHERE.

Syntaxe

SELECT colonnes FROM table WHERE condition;

Comparaisons simples

Égalité

SELECT * FROM porta_operateur WHERE code = '02';

Résultat : uniquement Digicel.

Différent de

SELECT * FROM porta_msisdn WHERE operateur_id_actuel != '02';

Tous les numéros qui ne sont PAS chez Digicel.

Supérieur / Inférieur

SELECT * FROM porta_portage WHERE date_portage > '2026-02-01';

Portages prévus après le 1er février 2026.

Combiner des conditions

AND — Les deux conditions doivent être vraies

SELECT * FROM porta_portage
WHERE etat_id_actuel = 2
  AND date_portage < '2026-02-10';

Portages « en cours » dont la date de portage est passée — potentiellement bloqués.

OR — Au moins une condition doit être vraie

SELECT * FROM porta_data
WHERE code_ticket = '1220'
   OR code_ticket = '7000';

Tous les tickets qui sont soit des refus (1220) soit des erreurs techniques (7000).

Combiner AND et OR avec des parenthèses

SELECT * FROM porta_data
WHERE code_ticket = '1220'
  AND (code_motif = 'R100' OR code_motif = 'R110');

Refus spécifiquement pour RIO invalide ou RIO expiré.

Opérateurs spéciaux

IN — Parmi une liste de valeurs

SELECT * FROM porta_data
WHERE code_motif IN ('R100', 'R110', 'R200', 'R210');

Équivalent à 4 conditions OR, mais plus lisible.

BETWEEN — Dans une plage

SELECT * FROM porta_fichier
WHERE date BETWEEN '2026-02-01' AND '2026-02-15';

LIKE — Recherche partielle (pattern matching)

SELECT * FROM porta_fichier
WHERE filename LIKE 'PNMDATA_02_%';

Tous les fichiers envoyés par Digicel (code 02). Le % remplace n'importe quel texte.

IS NULL / IS NOT NULL

SELECT * FROM porta_data WHERE code_motif IS NOT NULL;

Tickets qui ont un motif de refus (la colonne n'est pas vide).

Cas pratique PNM

Trouver tous les portages bloqués (en cours depuis plus de 5 jours) :

SELECT id, msisdn, date_portage, etat_id_actuel
FROM porta_portage
WHERE etat_id_actuel = 2
  AND date_portage < CURRENT_DATE - INTERVAL '5 days'
ORDER BY date_portage;
Exercices :
  1. Trouvez les numéros chez Orange : SELECT * FROM porta_msisdn WHERE operateur_id_actuel = '01'
  2. Trouvez les fichiers de type « sync » : SELECT * FROM porta_fichier WHERE type = 'sync'
  3. Trouvez les refus RIO invalide : SELECT * FROM porta_data WHERE code_ticket = '1220' AND code_motif = 'R100'
  4. Trouvez les jours fériés de mai 2026 : SELECT * FROM porta_ferryday WHERE ferryday BETWEEN '2026-05-01' AND '2026-05-31'
