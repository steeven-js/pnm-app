PortaDB — Base de données de portabilité

PortaDB est le système central de gestion de la portabilité chez Digicel Antilles-Guyane. C'est une base de données relationnelle qui stocke l'intégralité des informations relatives aux portages : tickets, statuts, historiques et tables de référence.

Rôle de PortaDB

PortaDB assure plusieurs fonctions critiques :

  • Référentiel des numéros portés : pour chaque MSISDN porté, PortaDB enregistre l'opérateur hébergeant actuel, l'opérateur attributaire, et la date du dernier portage
  • Gestion des tickets : création, stockage et suivi de l'état de chaque ticket PNM (1110, 1210, 1410, etc.)
  • Historique complet : traçabilité de toutes les opérations de portage (standard, inverse, restitution, annulation)
  • Source de données pour la génération des fichiers PNMDATA sortants et le traitement des fichiers entrants

Tables principales

PortaDB est structurée autour de plusieurs tables clés :

  • portages — Table centrale : un enregistrement par dossier de portage, contenant le MSISDN, les opérateurs impliqués (OPR, OPD, OPA), le statut courant, les dates clés (JD, JP)
  • tickets — Détail de chaque ticket émis ou reçu, avec type (1110, 1210...), horodatage, contenu, et statut de traitement
  • numeros_portes — Table de référence des numéros actuellement portés, mise à jour après chaque bascule effective
  • operateurs — Table de référence des opérateurs avec leurs codes (02, 15, 20), noms et paramètres sFTP
  • motifs_refus — Catalogue des codes de refus avec leurs libellés
  • historique_sync — Journal des synchronisations PNMSYNC (date, résultat, écarts détectés)

Hébergement

PortaDB est hébergée sur le serveur vmqproportawebdb01, une machine virtuelle dédiée dans l'infrastructure Digicel. Ce serveur héberge également l'interface web d'administration permettant aux équipes de consulter et gérer les dossiers de portage.

Relations avec PortaSync

PortaDB fonctionne en tandem avec PortaSync, le service de synchronisation :

  • PortaSync lit les tickets à envoyer dans PortaDB et génère les fichiers PNMDATA
  • PortaSync écrit dans PortaDB les tickets reçus des autres opérateurs via sFTP
  • PortaSync déclenche les mises à jour de statut dans PortaDB après chaque échange réussi

Relations avec l'ESB DataPower

L'ESB DataPower sert d'intermédiaire entre PortaDB et les systèmes internes :

  • Transmission des ordres de bascule HLR vers les équipements réseau (MOBI)
  • Remontée des résultats de bascule vers PortaDB
  • Interface avec le CRM pour les notifications de statut de portage

Maintenance

La maintenance de PortaDB inclut des sauvegardes quotidiennes, une purge périodique des dossiers anciens (archivage après 12 mois), et un monitoring des performances (temps de requête, espace disque). Les accès sont restreints aux équipes Applications et BSS via des comptes nominatifs.
