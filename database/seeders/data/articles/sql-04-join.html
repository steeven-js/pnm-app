JOIN — Croiser deux tables

Pourquoi joindre des tables ?

Les données PNM sont réparties dans 18 tables. Pour obtenir une vue complète, il faut croiser les informations. Par exemple, la table porta_portage contient un portage_etat (un nom d'état comme « enCours »), et c'est dans porta_etat qu'on trouve les détails de cet état.

JOIN permet de relier deux tables par une colonne commune.

INNER JOIN — La jointure classique

Retourne uniquement les lignes qui ont une correspondance dans les deux tables.

SELECT p.id, p.portage_msisdn, p.portage_date_souhaitee, e.etat_name AS etat
FROM porta_portage p
JOIN porta_etat e ON e.etat_name = p.portage_etat;

Décortiquons :

  • porta_portage p — on donne l'alias p à la table portage
  • JOIN porta_etat e — on joint la table etat avec l'alias e
  • ON e.etat_name = p.portage_etat — la condition de jointure (sur le nom de l'état)
  • e.etat_name AS etat — on affiche le nom de l'état

LEFT JOIN — Garder les lignes sans correspondance

LEFT JOIN garde toutes les lignes de la table de gauche, même si elles n'ont pas de correspondance à droite (les colonnes manquantes seront NULL).

Cas d'usage PNM : trouver les fichiers sans accusé de réception (ACR manquant) :

SELECT f.id, f.nom, f.date_creation, a.id AS ack_id
FROM porta_fichier f
LEFT JOIN porta_ack a ON a.file_name = f.nom
WHERE a.id IS NULL;

Explication :

  • On joint fichier avec ack via le nom de fichier
  • LEFT JOIN conserve les fichiers même sans ACK
  • WHERE a.id IS NULL filtre uniquement ceux qui n'ont PAS d'ACK

Joindre plusieurs tables

On peut chaîner les jointures pour obtenir une vue complète :

SELECT
    p.portage_msisdn,
    p.portage_date_souhaitee,
    p.portage_etat AS etat,
    d.operateur_id_donneur AS de,
    d.operateur_id_receveur AS vers,
    op_don.nom AS operateur_donneur,
    op_rec.nom AS operateur_receveur
FROM porta_portage p
JOIN porta_dossier d ON d.id = p.dossier_id
JOIN porta_operateur op_don ON op_don.code = d.operateur_id_donneur
JOIN porta_operateur op_rec ON op_rec.code = d.operateur_id_receveur
ORDER BY p.portage_date_souhaitee DESC
LIMIT 20;

Cette requête affiche les 20 derniers portages avec : numéro, date, état, et noms complets des opérateurs donneur et receveur.

Exercices :
  1. Affichez les portages avec le nom de l'état : 
    SELECT p.portage_msisdn, e.etat_name
FROM porta_portage p
JOIN porta_etat e ON e.etat_name = p.portage_etat
LIMIT 10;
  2. Trouvez les fichiers envoyés par Digicel avec le nom du destinataire : 
    SELECT f.nom, f.date_creation, o.nom AS destinataire
FROM porta_fichier f
JOIN porta_operateur o ON o.code = f.destinataire
WHERE f.expediteur = 2
LIMIT 10;
  3. Trouvez les fichiers sans ACR (anomalie) : 
    SELECT f.id, f.nom, f.date_creation
FROM porta_fichier f
LEFT JOIN porta_ack a ON a.file_name = f.nom
WHERE a.id IS NULL;
