JOIN — Croiser deux tables

Pourquoi joindre des tables ?

Les données PNM sont réparties dans 18 tables. Pour obtenir une vue complète, il faut croiser les informations. Par exemple, la table porta_portage contient un etat_id_actuel (un numéro), mais c'est dans porta_etat qu'on trouve le libellé lisible (« En cours entrante »).

JOIN permet de relier deux tables par une colonne commune.

INNER JOIN — La jointure classique

Retourne uniquement les lignes qui ont une correspondance dans les deux tables.

SELECT p.id, p.msisdn, p.date_portage, e.label AS etat
FROM porta_portage p
JOIN porta_etat e ON e.id = p.etat_id_actuel;

Décortiquons :

  • porta_portage p — on donne l'alias p à la table portage
  • JOIN porta_etat e — on joint la table etat avec l'alias e
  • ON e.id = p.etat_id_actuel — la condition de jointure
  • e.label AS etat — on affiche le libellé de l'état

LEFT JOIN — Garder les lignes sans correspondance

LEFT JOIN garde toutes les lignes de la table de gauche, même si elles n'ont pas de correspondance à droite (les colonnes manquantes seront NULL).

Cas d'usage PNM : trouver les fichiers sans accusé de réception (ACR manquant) :

SELECT f.id, f.filename, f.date, a.id AS ack_id
FROM porta_fichier f
LEFT JOIN porta_ack a ON a.fichier_id = f.id
WHERE a.id IS NULL;

Explication :

  • On joint fichier avec ack
  • LEFT JOIN conserve les fichiers même sans ACK
  • WHERE a.id IS NULL filtre uniquement ceux qui n'ont PAS d'ACK

Joindre plusieurs tables

On peut chaîner les jointures pour obtenir une vue complète :

SELECT
    p.msisdn,
    p.date_portage,
    e.label AS etat,
    d.operateur_id_origine AS de,
    d.operateur_id_destination AS vers,
    op_orig.nom AS operateur_origine,
    op_dest.nom AS operateur_destination
FROM porta_portage p
JOIN porta_etat e ON e.id = p.etat_id_actuel
JOIN porta_dossier d ON d.id = p.dossier_id
JOIN porta_operateur op_orig ON op_orig.code = d.operateur_id_origine
JOIN porta_operateur op_dest ON op_dest.code = d.operateur_id_destination
ORDER BY p.date_portage DESC
LIMIT 20;

Cette requête affiche les 20 derniers portages avec : numéro, date, état lisible, et noms complets des opérateurs d'origine et de destination.

Exercices :
  1. Affichez les portages avec le nom de l'état : 
    SELECT p.msisdn, e.label
FROM porta_portage p
JOIN porta_etat e ON e.id = p.etat_id_actuel
LIMIT 10;
  2. Trouvez les fichiers envoyés par Digicel avec le nom du destinataire : 
    SELECT f.filename, f.date, o.nom AS destinataire
FROM porta_fichier f
JOIN porta_operateur o ON o.code = f.destinataire
WHERE f.expediteur = '02'
LIMIT 10;
  3. Trouvez les fichiers sans ACR (anomalie) : 
    SELECT f.id, f.filename, f.date
FROM porta_fichier f
LEFT JOIN porta_ack a ON a.fichier_id = f.id
WHERE a.id IS NULL;
