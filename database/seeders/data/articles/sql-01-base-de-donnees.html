C'est quoi une base de données ?

Introduction

Une base de données est un système organisé pour stocker, gérer et retrouver des informations. Dans le contexte de la portabilité des numéros mobiles (PNM), la base de données PORTA est le cœur du système : elle enregistre chaque demande de portage, chaque échange de fichier, et chaque transition d'état d'un numéro.

Pensez à la base de données comme un classeur géant avec des onglets (les tables), et dans chaque onglet, des lignes et des colonnes organisant les informations.

Les tables : les onglets du classeur

PORTA contient 18 tables, réparties en deux catégories :

Tables de référence (les « dictionnaires »)

Ces tables contiennent des données qui changent rarement. Elles servent de référence pour les données opérationnelles.

Comment les tables sont liées entre elles

Les tables ne sont pas isolées. Elles sont reliées par des clés étrangères (foreign keys). Par exemple :

  • Un portage référence un dossier (par dossier_id)
  • Un portage référence un etat (par portage_etat)
  • Un fichier référence deux operateur (expéditeur et destinataire)
  • Un ticket portage_data référence un portage et un code_ticket

Ces liens permettent de naviguer d'une information à l'autre. Par exemple, à partir d'un numéro de téléphone, on peut retrouver son portage, puis son dossier, puis les fichiers échangés, puis les accusés de réception.

Le langage SQL

SQL (Structured Query Language) est le langage utilisé pour interroger une base de données. C'est comme poser des questions au classeur :

  • « Montre-moi tous les opérateurs » → SELECT * FROM porta_operateur
  • « Combien de portages sont en cours ? » → SELECT COUNT(*) FROM porta_portage WHERE portage_etat = 'enCours'

Dans les articles suivants, vous apprendrez pas à pas les commandes SQL pour interroger ces tables et mener des enquêtes sur les anomalies PNM.
