Impact de la portabilité sur le routage réseau

La portabilité des numéros mobiles modifie en profondeur la manière dont les appels sont routés à travers le réseau. Sans portabilité, le préfixe d'un numéro suffit à identifier l'opérateur. Avec la PNM, un mécanisme de consultation supplémentaire est nécessaire.

Routage classique vs. routage avec portabilité

Dans un réseau sans portabilité, le routage est simple :

  • Le MSC analyse le préfixe du numéro appelé (ex : 0690 = Digicel Guadeloupe)
  • L'appel est acheminé directement vers l'opérateur correspondant au préfixe

Avec la portabilité, cette logique ne fonctionne plus car un numéro 0690XXXXXX (préfixe historiquement Digicel) peut être hébergé chez Orange ou SFR/Only. Le réseau doit donc interroger une base de données avant de router l'appel.

Mécanisme de routage des numéros portés

Lorsqu'un appel est émis vers un numéro potentiellement porté :

  • Le MSC de l'opérateur appelant interroge son HLR
  • Le HLR consulte la table de routage PNM (alimentée par MOBI) pour vérifier si le numéro a été porté
  • Si le numéro est porté, le HLR retourne un préfixe de routage spécifique indiquant l'opérateur hébergeant
  • Le MSC redirige l'appel vers l'opérateur hébergeant via l'interconnexion appropriée
  • L'opérateur hébergeant achemine l'appel jusqu'à l'abonné sur son réseau

Mises à jour du HLR lors d'un portage

Lors de la bascule technique (jour JP), plusieurs mises à jour sont effectuées :

  • HLR de l'OPR : création du profil abonné avec le MSISDN porté, association avec la nouvelle IMSI/SIM
  • HLR de l'OPD : suppression du profil abonné, désactivation de l'ancienne SIM
  • Tables de routage MOBI : mise à jour de l'association MSISDN → opérateur hébergeant, diffusée à tous les opérateurs

Rôle de MOBI

MOBI est le système de gestion des tables de routage PNM. Il reçoit les commandes de mise à jour via l'ESB DataPower :

  • EMA (Enregistrement MSISDN Ajout) : ajoute un numéro porté dans les tables de routage avec le nouvel opérateur hébergeant
  • EMM (Enregistrement MSISDN Modification) : modifie le routage d'un numéro déjà porté (portage inverse)
  • EMS (Enregistrement MSISDN Suppression) : retire un numéro des tables de portage (restitution à l'OPA)

Impacts sur la qualité de service

La portabilité peut engendrer un léger surcoût en temps de routage (quelques millisecondes pour la consultation HLR). En cas de désynchronisation des tables de routage entre opérateurs, un numéro porté peut temporairement être injoignable. C'est pourquoi la synchronisation hebdomadaire (PNMSYNC) est essentielle pour détecter et corriger ces divergences.
