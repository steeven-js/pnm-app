Enquête : refus de portage

Le scénario

Plusieurs scénarios peuvent déclencher cette enquête :

  • Email automatique : « 8 refus de portage (1220) détectés pour RIO incorrect »
  • Email automatique : « 3 tickets erreur 7000 détectés »
  • Demande orale : « Le client dit que son portage a été refusé, vérifie pourquoi »

Comprendre les codes de refus

D'abord, consultez les codes existants :

SELECT * FROM porta_code_reponse
WHERE type = 'eligibilite'
ORDER BY code;

Étape 1 : Trouver tous les refus

SELECT d.msisdn, d.rio, d.code_motif, cr.label AS motif,
       f.date, f.expediteur, f.destinataire
FROM porta_data d
JOIN porta_code_reponse cr ON cr.code = d.code_motif
JOIN porta_fichier f ON f.id = d.fichier_id
WHERE d.code_ticket = '1220'
ORDER BY f.date DESC;

Étape 2 : Refus pour RIO invalide spécifiquement

SELECT d.msisdn, d.rio, cr.label AS motif, f.date
FROM porta_data d
JOIN porta_code_reponse cr ON cr.code = d.code_motif
JOIN porta_fichier f ON f.id = d.fichier_id
WHERE d.code_ticket = '1220'
  AND d.code_motif IN ('R100', 'R110')
ORDER BY f.date DESC;

Vérifiez la colonne rio : si elle contient des valeurs comme « XX0000000000 », le RIO est clairement invalide.

Étape 3 : Trouver les erreurs 7000

Le code 7000 indique un champ obligatoire manquant dans un ticket :

SELECT d.id, d.msisdn, d.rio, d.id_portage, d.code_ticket,
       f.filename, f.date
FROM porta_data d
JOIN porta_fichier f ON f.id = d.fichier_id
WHERE d.code_ticket = '7000'
ORDER BY f.date DESC;

Examinez les colonnes : un rio à NULL ou un id_portage manquant est souvent la cause.

Étape 4 : Statistiques des refus par période

SELECT
    f.date,
    d.code_motif,
    cr.label AS motif,
    COUNT(*) AS total
FROM porta_data d
JOIN porta_code_reponse cr ON cr.code = d.code_motif
JOIN porta_fichier f ON f.id = d.fichier_id
WHERE d.code_ticket = '1220'
GROUP BY f.date, d.code_motif, cr.label
ORDER BY f.date DESC, total DESC;

Permet de détecter un pic de refus sur un jour particulier.

Étape 5 : Taux de refus par opérateur

SELECT
    o.nom AS operateur_donneur,
    COUNT(*) FILTER (WHERE d.code_ticket = '1220') AS refus,
    COUNT(*) FILTER (WHERE d.code_ticket = '1210') AS acceptes,
    ROUND(
        COUNT(*) FILTER (WHERE d.code_ticket = '1220') * 100.0 /
        NULLIF(COUNT(*) FILTER (WHERE d.code_ticket IN ('1210', '1220')), 0),
        1
    ) AS taux_refus_pct
FROM porta_data d
JOIN porta_fichier f ON f.id = d.fichier_id
JOIN porta_operateur o ON o.code = f.expediteur
WHERE d.code_ticket IN ('1210', '1220')
GROUP BY o.nom
ORDER BY taux_refus_pct DESC;

Note avancée : FILTER (WHERE ...) est une syntaxe PostgreSQL qui permet de compter conditionnellement. NULLIF évite la division par zéro.

  • 3 tickets avec code 7000 (erreur technique)
