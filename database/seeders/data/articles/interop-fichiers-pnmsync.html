Fichiers PNMSYNC — Synchronisation hebdomadaire

Les fichiers PNMSYNC sont le mécanisme de synchronisation inter-opérateurs permettant de vérifier que tous les opérateurs ont une vision cohérente des numéros portés. Ce processus, exécuté chaque dimanche, est essentiel pour détecter et corriger les divergences.

Objectif de la synchronisation

Au fil des vacations quotidiennes, des incohérences peuvent apparaître entre les bases de données des opérateurs :

  • Un fichier PNMDATA perdu ou corrompu pendant le transfert sFTP
  • Un ticket traité côté émetteur mais pas côté récepteur
  • Une bascule HLR effectuée sans confirmation dans PortaDB
  • Une erreur de traitement silencieuse dans un script

La synchronisation PNMSYNC permet de comparer les bases de chaque opérateur et d'identifier ces divergences avant qu'elles n'impactent le routage des appels.

Processus du dimanche

La synchronisation suit un processus en plusieurs étapes :

1. Extraction (dimanche matin) :

  • Chaque opérateur extrait de sa base la liste complète des MSISDN portés
  • Pour chaque MSISDN, le fichier contient : le numéro, l'opérateur hébergeant actuel, l'opérateur attributaire, la date du dernier portage
  • Le fichier est nommé selon la convention : PNMSYNC_[OPE]_[DATE].dat

2. Échange (dimanche matin) :

  • Chaque opérateur dépose son fichier PNMSYNC sur les serveurs sFTP des deux autres opérateurs
  • Chaque opérateur collecte les fichiers PNMSYNC des deux autres

3. Comparaison (dimanche après-midi) :

  • PortaSync compare la liste reçue de chaque opérateur avec sa propre base PortaDB
  • Les MSISDN présents dans une base mais absents de l'autre sont identifiés comme divergences
  • Les MSISDN avec un opérateur hébergeant différent sont signalés comme incohérences de routage

4. Rapport et correction :

  • Un rapport de divergences est généré automatiquement
  • Les divergences mineures (nombre limité) sont corrigées automatiquement dans PortaDB
  • Les divergences majeures ou systématiques sont escaladées au comité technique GPMAG

Contenu d'un fichier PNMSYNC

Chaque enregistrement du fichier PNMSYNC contient :

  • MSISDN : numéro porté au format international
  • OPH : code de l'opérateur hébergeant actuel
  • OPA : code de l'opérateur attributaire
  • DATE_PORTAGE : date du dernier portage effectif
  • NUM_DOSSIER : référence du dernier dossier de portage

Importance opérationnelle

Un taux de divergence supérieur à 0.1% est considéré comme anormal et déclenche une investigation. Les divergences non résolues peuvent entraîner des appels mal routés (l'appelant entend un message d'erreur ou est dirigé vers le mauvais opérateur), d'où l'importance critique de ce processus hebdomadaire.
