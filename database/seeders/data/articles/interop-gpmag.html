GPMAG — Structure et gouvernance inter-opérateurs

Le GPMAG (Groupement de Portabilité des Numéros Mobiles Antilles-Guyane) est l'instance de coordination inter-opérateurs qui régit la portabilité mobile dans les départements français d'Amérique. Il regroupe les trois opérateurs mobiles de la zone et définit les règles communes de fonctionnement.

Les opérateurs membres

Six opérateurs composent le GPMAG, chacun identifié par un code opérateur unique utilisé dans les fichiers PNMDATA, les tickets PNM et le RIO :

  • Orange Caraïbe — Code 01 : filiale du groupe Orange dédiée à la zone Antilles-Guyane. Infrastructure complète avec couverture étendue.
  • Digicel — Code 02 : opérateur historique des Antilles-Guyane, présent en Guadeloupe, Martinique et Guyane. Réseau 2G/3G/4G.
  • SFR / Only — Code 03 : opérateur opérant sous les marques SFR et Only selon les territoires.
  • Dauphin Télécom — Code 04 : opérateur local présent principalement à Saint-Martin et Saint-Barthélemy.
  • UTS (United Telecommunication Services) — Code 05 : opérateur actif sur la partie néerlandaise de Saint-Martin avec des services transfrontaliers.
  • Free Caraïbe — Code 06 : filiale du groupe Iliad pour la zone Antilles-Guyane.

Ces codes opérateurs (01 à 06) sont utilisés dans le RIO, les tickets PNM, les fichiers PNMDATA et les logs PortaSync pour identifier sans ambiguïté chaque acteur.

Liaisons sFTP

Les échanges de fichiers PNM entre opérateurs s'effectuent via des liaisons sFTP sécurisées point à point :

  • Chaque opérateur dispose d'un serveur sFTP dédié aux échanges PNM
  • Les connexions sont authentifiées par certificats SSL et clés SSH
  • Chaque paire d'opérateurs maintient une liaison bidirectionnelle (ex : Digicel ↔ Orange, Digicel ↔ SFR, Orange ↔ SFR)
  • Les répertoires de dépôt et de collecte suivent une convention de nommage standardisée : /pnm/[code_emetteur]/[code_destinataire]/

Les fichiers déposés sont les PNMDATA (tickets), ACR (accusés de réception), ERR (erreurs), et PNMSYNC (synchronisation).

Gouvernance

Le GPMAG fonctionne selon une gouvernance collégiale :

  • Comité technique : réunion mensuelle entre les référents techniques PNM de chaque opérateur. Traitement des incidents inter-opérateurs, analyse des statistiques de portage, planification des évolutions.
  • Comité de pilotage : réunion trimestrielle au niveau direction. Validation des évolutions majeures du protocole, arbitrage des litiges non résolus au niveau technique.
  • Spécifications communes : le GPMAG maintient un document de spécifications techniques PNM V3 partagé, qui fait foi en cas de divergence d'interprétation.

Contacts techniques GPMAG

Chaque opérateur du GPMAG désigne des référents techniques PNM joignables pour le traitement des incidents et la coordination inter-opérateurs :

  • Digicel : équipe PNM interne — via canal Teams PNM et email générique PNM
  • Orange Caraïbe : référent technique portabilité — contact via email générique portabilité Orange AG
  • SFR / Only : équipe technique PNM — contact via email générique portabilité SFR AG
  • Dauphin Télécom : référent technique — contact via email opérateur
  • UTS : référent technique — contact via email opérateur
  • Free Caraïbe : référent technique — contact via email opérateur

Les coordonnées nominatives et emails génériques sont disponibles dans le répertoire GPMAG partagé (SharePoint PNM). En cas d'incident critique, privilégier le contact téléphonique direct suivi d'un email de confirmation.

Obligations des membres

Chaque opérateur membre du GPMAG s'engage à :

  • Maintenir ses systèmes PNM opérationnels 7j/7
  • Respecter les délais de traitement des tickets (réponse sous 24h ouvrées)
  • Participer aux synchronisations hebdomadaires (PNMSYNC du dimanche)
  • Signaler tout incident impactant la portabilité dans un délai de 2 heures
  • Fournir des statistiques mensuelles de portage à l'ARCEP
