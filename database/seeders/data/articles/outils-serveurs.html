Inventaire des serveurs PNM

L'infrastructure de portabilité de Digicel repose sur un ensemble de serveurs dédiés, chacun hébergeant des composants spécifiques de la chaîne PNM. Connaître ces serveurs et leur rôle est indispensable pour le diagnostic et la maintenance.

vmqproportawebdb01 — Serveur PortaDB

Serveur principal de la plateforme de portabilité.

  • Rôle : héberge la base de données PortaDB et l'interface web d'administration
  • Composants :
    • Base de données relationnelle PortaDB (stockage des tickets, numéros portés, historiques)
    • Application web de gestion (consultation et pilotage des dossiers de portage)
    • Scripts de bascule (TraitementBascule.sh, BasculeVerification.sh)
    • Scripts de restitution et d'export CSV
  • OS : Linux (Red Hat Enterprise Linux)
  • Sauvegarde : backup quotidien de la base de données à 02h00, rétention 30 jours
  • Accès : restreint aux équipes Applications et BSS via SSH (authentification par clé)
  • Supervision : monitoring CPU, RAM, disque, et processus base de données

vmqproportasync01 — Serveur PortaSync

Serveur dédié aux échanges inter-opérateurs.

  • Rôle : héberge le service PortaSync et gère les transferts sFTP
  • Utilisateur dédié : porta_pnmv3
  • Dossier PortaSync : /home/porta_pnmv3/PortaSync
  • Logs : /home/porta_pnmv3/PortaSync/log (EmaExtracter.log, EmmExtracter.log, PnmDataManager.log, PnmAckManager.log)
  • Composants :
    • Service PortaSync (génération et traitement des fichiers PNMDATA)
    • Serveur sFTP local (réception des fichiers des opérateurs partenaires)
    • Scripts PHP PortaSync (EmaExtracter, EmmExtracter, PnmDataManager, PnmSyncAckManager)
    • Scripts d'export/import (pnm-import.sh, portasync-export.sh)
    • Scripts de synchronisation PNMSYNC
    • Répertoire d'archivage des fichiers échangés
  • Web services hébergés :
    • PortaWs : http://172.24.119.72:8080/PortaWs/ — Portail de consultation des portages en cours et services DAPI (DigicelFwiPortaWs4Esb, DigicelFwiPortaWs4PortaSync)
    • PortaWebUi : http://172.24.119.71:8080/PortaWebUi/ — Interface d'administration PNM et service DigicelFwiPortaUiWs4Esb
  • OS : Linux (Red Hat Enterprise Linux)
  • Réseau : interface dédiée pour les flux sFTP inter-opérateurs, firewallé avec accès restreint aux IP des partenaires GPMAG
  • Stockage : espace dédié pour l'archivage des fichiers PNMDATA (rétention 12 mois)
  • Supervision : monitoring des connexions sFTP, espace disque, et processus PortaSync

BTCTF — Bureau Technique Central

Plateforme externe nationale de référence pour la portabilité.

  • Rôle : référentiel national des numéros portés, géré au niveau métropolitain
  • Interaction : PortaSync communique avec le BTCTF pour les synchronisations nationales et les cas de portage inter-zones
  • Connexion : liaison sécurisée dédiée depuis vmqproportasync01
  • Fréquence : échanges quotidiens pour les mises à jour incrémentales, synchronisation complète hebdomadaire

Réseau inter-serveurs

Les serveurs PNM communiquent via le réseau interne Digicel :

  • vmqproportawebdb01 ↔ vmqproportasync01 : connexion base de données sécurisée (port dédié, authentification applicative)
  • vmqproportawebdb01 ↔ ESB DataPower : flux web services pour les commandes de bascule HLR
  • vmqproportasync01 ↔ sFTP partenaires : flux sFTP sortants vers Orange et SFR/Only

Plan de continuité

En cas de panne d'un serveur, les procédures de disaster recovery prévoient une restauration à partir des sauvegardes sous 4 heures. Les vacations impactées sont reportées et les opérateurs partenaires sont notifiés conformément aux engagements GPMAG.
