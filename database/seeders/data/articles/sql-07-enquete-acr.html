Enquête : fichiers et ACR manquants

Le scénario

Vous recevez un email automatique : « Fichier PNMDATA_02_01_20260210_V10_0042.txt envoyé il y a plus de 60 minutes sans accusé de réception. »

Un ACR manquant signifie que le fichier envoyé n'a pas été acquitté par le destinataire. C'est potentiellement un problème de transmission sFTP, de traitement chez l'opérateur distant, ou de fichier corrompu.

Étape 1 : Vérifier le fichier en question

SELECT f.id, f.nom, f.expediteur, f.destinataire,
       f.date_creation, f.taille, f.checksum
FROM porta_fichier f
WHERE f.nom LIKE '%20260210_V10%'
  AND f.expediteur = 2;

Étape 2 : Trouver TOUS les fichiers sans ACR

C'est LE cas d'usage du LEFT JOIN :

SELECT f.id, f.nom, f.date_creation, f.expediteur, f.destinataire
FROM porta_fichier f
LEFT JOIN porta_ack a ON a.file_name = f.nom
WHERE a.id IS NULL
  AND f.type = 'data'
ORDER BY f.date_creation DESC;

Explication :

  • LEFT JOIN porta_ack : on garde tous les fichiers, même sans ACK. La jointure se fait sur file_name = nom
  • WHERE a.id IS NULL : on filtre ceux qui n'ont vraiment aucun ACK
  • AND f.type = 'data' : on ne regarde que les PNMDATA (pas les PNMSYNC)

Étape 3 : Vérifier les ACR en erreur

Certains fichiers ont reçu un ACR mais c'est un ERR (fichier d'erreur) :

SELECT f.nom, f.date_creation, a.type AS type_ack, a.content
FROM porta_fichier f
JOIN porta_ack a ON a.file_name = f.nom
WHERE a.type = 'ERR'
ORDER BY f.date_creation DESC;

Étape 4 : Compter les ACR manquants par opérateur

SELECT o.nom AS destinataire, COUNT(*) AS acr_manquants
FROM porta_fichier f
LEFT JOIN porta_ack a ON a.file_name = f.nom
JOIN porta_operateur o ON o.code = f.destinataire
WHERE a.id IS NULL
  AND f.type = 'data'
GROUP BY o.nom
ORDER BY acr_manquants DESC;

Identifie quel opérateur a le plus de problèmes de réception.

Étape 5 : Impact sur les portages

Un fichier sans ACR contient des tickets. Quels portages sont impactés ?

SELECT pd.id, pd.code_ticket, pd.code_reponse, pd.etat,
       p.portage_msisdn, f.nom AS fichier
FROM porta_portage_data pd
JOIN porta_portage p ON p.id = pd.id_portage
JOIN porta_fichier f ON f.nom IN (
    SELECT f2.nom FROM porta_fichier f2
    LEFT JOIN porta_ack a ON a.file_name = f2.nom
    WHERE a.id IS NULL
)
ORDER BY pd.date DESC;

On retrouve les tickets concernés par des fichiers non acquittés, via porta_portage_data.

Temps de réponse des ACR

Vérifier le délai moyen entre l'envoi du fichier et la réception de l'ACR :

SELECT
    o.nom AS destinataire,
    AVG(EXTRACT(EPOCH FROM (a.date - f.date_creation)) / 60) AS delai_moyen_minutes
FROM porta_fichier f
JOIN porta_ack a ON a.file_name = f.nom
JOIN porta_operateur o ON o.code = f.destinataire
WHERE f.type = 'data'
GROUP BY o.nom
ORDER BY delai_moyen_minutes DESC;
Anomalie à trouver : Dans les données de test, 5 fichiers récents n'ont pas reçu d'ACR. Utilisez la requête LEFT JOIN pour les identifier, puis vérifiez quels tickets ils contenaient.
