Catalogue des scripts PNM

L'exploitation quotidienne de la portabilité repose sur un ensemble de scripts automatisés exécutés sur les serveurs PortaDB et PortaSync. Ce catalogue recense les scripts principaux, leur fonction, leur ordonnancement et leur localisation.

Scripts de bascule

TraitementBascule.sh

  • Fonction : script principal de bascule. Identifie les portages à basculer au jour JP, envoie les commandes EMA/EMM vers MOBI via l'ESB DataPower, vérifie les retours, et met à jour PortaDB.
  • Ordonnancement : exécuté quotidiennement lors de la vacation V10 (10h00), après import des fichiers PNMDATA.
  • Serveur : vmqproportawebdb01
  • Chemin : /opt/portadb/scripts/TraitementBascule.sh
  • Criticité : haute — un échec retarde les portages du jour.

BasculeVerification.sh

  • Fonction : vérifie le résultat des bascules exécutées par TraitementBascule.sh. Contrôle que les commandes EMA/EMM ont été acquittées positivement par MOBI et que le HLR a bien été mis à jour.
  • Ordonnancement : exécuté 30 minutes après TraitementBascule.sh.
  • Serveur : vmqproportawebdb01
  • Chemin : /opt/portadb/scripts/BasculeVerification.sh

Scripts d'export/import

PortaDB-export-csv.sh

  • Fonction : exporte les données de PortaDB au format CSV pour les besoins de reporting, d'audit ou d'analyse. Génère des extractions par période, par opérateur, ou par statut.
  • Ordonnancement : à la demande, ou quotidiennement à 23h00 pour les rapports automatiques.
  • Serveur : vmqproportawebdb01
  • Chemin : /opt/portadb/scripts/PortaDB-export-csv.sh

pnm-import.sh

  • Fonction : importe les tickets PNM reçus des autres opérateurs dans PortaDB. Parse les fichiers PNMDATA, valide chaque ticket, et insère les données dans les tables correspondantes.
  • Ordonnancement : 3 fois par jour, à chaque vacation (V10, V14, V19), après la collecte sFTP.
  • Serveur : vmqproportasync01
  • Chemin : /opt/portasync/scripts/pnm-import.sh

Scripts de restitution

RestitutionProcess.sh

  • Fonction : gère le processus de restitution des numéros portés à leur opérateur attributaire. Identifie les numéros résiliés chez Digicel qui doivent être restitués, génère les tickets 3110, et suit le cycle de restitution.
  • Ordonnancement : quotidien à 08h00, avant la vacation V10.
  • Serveur : vmqproportawebdb01
  • Chemin : /opt/portadb/scripts/RestitutionProcess.sh

RestitutionCleanup.sh

  • Fonction : nettoie les restitutions terminées, archive les dossiers clôturés, met à jour la table numeros_portes pour retirer les numéros restitués.
  • Ordonnancement : hebdomadaire, dimanche à 22h00.
  • Serveur : vmqproportawebdb01
  • Chemin : /opt/portadb/scripts/RestitutionCleanup.sh

Scripts de synchronisation

pnmsync-generate.sh

  • Fonction : génère le fichier PNMSYNC hebdomadaire contenant la liste complète des numéros portés dans notre base.
  • Ordonnancement : dimanche à 07h00.
  • Serveur : vmqproportasync01
  • Chemin : /opt/portasync/scripts/pnmsync-generate.sh

pnmsync-compare.sh

  • Fonction : compare les fichiers PNMSYNC reçus des autres opérateurs avec notre base PortaDB et génère le rapport de divergences.
  • Ordonnancement : dimanche à 14h00, après collecte des fichiers PNMSYNC partenaires.
  • Serveur : vmqproportasync01
  • Chemin : /opt/portasync/scripts/pnmsync-compare.sh

Scripts PortaSync (PHP)

En plus des scripts shell, PortaSync utilise des scripts PHP exécutés par cron sur vmqproportasync01 (utilisateur porta_pnmv3). Ces scripts gèrent les étapes clés du workflow quotidien :

EmaExtracter.php

  • Fonction : extraction des bascules EMA depuis PortaDB pour chacun des 6 opérateurs. Prépare les données de bascule (ajout MSISDN dans les tables de routage).
  • Ordonnancement : cron à 9h00
  • Chemin : /home/porta_pnmv3/PortaSync/
  • Log : EmaExtracter.log — doit afficher "Check success" pour les 6 opérateurs

EmmExtracter.php

  • Fonction : extraction des valorisations EMM depuis PortaDB pour chacun des 6 opérateurs. Prépare les données de modification d'opérateur hébergeant.
  • Ordonnancement : cron à 9h01
  • Chemin : /home/porta_pnmv3/PortaSync/
  • Log : EmmExtracter.log — doit afficher "Check success" pour les 6 opérateurs

PnmDataManager.php

  • Fonction : génération des fichiers PNMDATA à partir des données extraites. Produit un fichier par opérateur (01 à 06) contenant les tickets à transmettre.
  • Ordonnancement : cron à 10h00
  • Chemin : /home/porta_pnmv3/PortaSync/
  • Log : PnmDataManager.log — indique le nombre de tickets par fichier généré

PnmSyncAckManager.php

  • Fonction : vérification des acquittements AR SYNC reçus des opérateurs partenaires. Contrôle que chaque fichier PNMDATA envoyé a bien reçu son accusé de réception.
  • Ordonnancement : cron à 10h40
  • Chemin : /home/porta_pnmv3/PortaSync/
  • Log : PnmAckManager.log — liste les AR reçus et manquants

Bonnes pratiques

Tous les scripts sont ordonnancés via crontab et produisent des logs dans /var/log/ (scripts shell) ou /home/porta_pnmv3/PortaSync/log (scripts PHP) sur leur serveur respectif. En cas de modification d'un script, il est impératif de tester en environnement de pré-production et de notifier l'équipe via le canal PNM avant mise en production.
