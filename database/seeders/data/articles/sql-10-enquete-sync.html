Enquête : conflits de synchronisation

Le scénario

Chaque dimanche, les opérateurs échangent des fichiers PNMSYNC contenant la liste complète des MSISDN portés. La comparaison entre ces listes peut révéler des conflits : un numéro qui apparaît chez deux opérateurs différents, ou un numéro porté qui n'est pas dans la liste de l'opérateur receveur.

Alertes typiques :

  • Rapport hebdomadaire : « 3 conflits SYNC non résolus »
  • Anomalie réseau : « Le client X est routé vers Orange alors qu'il est chez Digicel »

Étape 1 : Trouver les conflits ouverts

SELECT ss.id, s.msisdn, s.date_portage,
       o.nom AS operateur_receveur,
       ss.commentaire, ss.created_at
FROM porta_sync_status ss
JOIN porta_sync s ON s.id = ss.sync_id
JOIN porta_operateur o ON o.code = s.operateur_receveur
WHERE ss.is_conflict = true
ORDER BY ss.created_at DESC;

Étape 2 : Vérifier l'historique du MSISDN en conflit

Pour chaque numéro en conflit, on vérifie son historique complet :

SELECT mh.msisdn, o.nom AS operateur, mh.date_debut, mh.date_fin
FROM porta_msisdn_historique mh
JOIN porta_operateur o ON o.code = mh.operateur_id
WHERE mh.msisdn = '0690XXXXXX'  -- remplacer par le MSISDN en conflit
ORDER BY mh.date_debut;

Étape 3 : Détecter les MSISDN avec plusieurs entrées actives

Un numéro ne devrait avoir qu'une seule entrée active (sans date_fin) dans l'historique :

SELECT mh.msisdn, COUNT(*) AS nb_entrees_actives
FROM porta_msisdn_historique mh
WHERE mh.date_fin IS NULL
GROUP BY mh.msisdn
HAVING COUNT(*) > 1;

Si cette requête retourne des résultats, c'est une anomalie grave : un numéro est enregistré comme actif chez plusieurs opérateurs en même temps.

Étape 4 : Détail des doublons

SELECT mh.msisdn, o.nom AS operateur, mh.date_debut, mh.portage_id
FROM porta_msisdn_historique mh
JOIN porta_operateur o ON o.code = mh.operateur_id
WHERE mh.date_fin IS NULL
  AND mh.msisdn IN (
      SELECT msisdn FROM porta_msisdn_historique
      WHERE date_fin IS NULL
      GROUP BY msisdn
      HAVING COUNT(*) > 1
  )
ORDER BY mh.msisdn, mh.date_debut;

Note : C'est une sous-requête (un SELECT dans un autre SELECT). Le SELECT interne trouve les MSISDN en double, puis le SELECT externe affiche les détails.

Étape 5 : Fichiers SYNC récents

SELECT f.filename, f.date,
       o_exp.nom AS expediteur,
       o_dest.nom AS destinataire,
       COUNT(s.id) AS nb_msisdn
FROM porta_fichier f
JOIN porta_operateur o_exp ON o_exp.code = f.expediteur
JOIN porta_operateur o_dest ON o_dest.code = f.destinataire
LEFT JOIN porta_sync s ON s.fichier_id = f.id
WHERE f.type = 'sync'
GROUP BY f.id, f.filename, f.date, o_exp.nom, o_dest.nom
ORDER BY f.date DESC
LIMIT 20;

Étape 6 : Croiser SYNC et portages

Vérifier qu'un MSISDN signalé en conflit a bien un portage correspondant :

SELECT
    s.msisdn,
    s.date_portage AS date_sync,
    p.date_portage AS date_portage_reel,
    e.label AS etat_portage,
    CASE
        WHEN p.id IS NULL THEN 'PAS DE PORTAGE TROUVE'
        WHEN e.classe = 'cloture' THEN 'OK - Cloture'
        ELSE 'ATTENTION - ' || e.label
    END AS diagnostic
FROM porta_sync s
JOIN porta_sync_status ss ON ss.sync_id = s.id
LEFT JOIN porta_portage p ON p.msisdn = s.msisdn
LEFT JOIN porta_etat e ON e.id = p.etat_id_actuel
WHERE ss.is_conflict = true
ORDER BY s.msisdn;
