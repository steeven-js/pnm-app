Enquête : conflits de synchronisation

Le scénario

Chaque dimanche, les opérateurs échangent des fichiers PNMSYNC contenant la liste complète des MSISDN portés. La comparaison entre ces listes peut révéler des conflits : un numéro qui apparaît chez deux opérateurs différents, ou un numéro porté qui n'est pas dans la liste de l'opérateur receveur.

Alertes typiques :

  • Rapport hebdomadaire : « 3 conflits SYNC non résolus »
  • Anomalie réseau : « Le client X est routé vers Orange alors qu'il est chez Digicel »

Étape 1 : Trouver les conflits ouverts

SELECT ss.id, ss.statut_name, ss.statut_code, ss.commentaire,
       s.msisdn, s.date,
       o.nom AS operateur
FROM porta_sync_status ss
JOIN porta_sync s ON s.sync_status = ss.id
JOIN porta_operateur o ON o.code = s.operateur_id
WHERE ss.statut_name = 'LOCKED'
ORDER BY s.date DESC;

Étape 2 : Vérifier l'historique du MSISDN en conflit

Pour chaque numéro en conflit, on vérifie son historique complet :

SELECT mh.msisdn, mh.date
FROM porta_msisdn_historique mh
WHERE mh.msisdn = '0690XXXXXX'  -- remplacer par le MSISDN en conflit
ORDER BY mh.date;

Étape 3 : Détecter les MSISDN avec plusieurs entrées

Un numéro ne devrait apparaître qu'un nombre limité de fois dans l'historique :

SELECT mh.msisdn, COUNT(*) AS nb_entrees
FROM porta_msisdn_historique mh
GROUP BY mh.msisdn
HAVING COUNT(*) > 2;

Si cette requête retourne des résultats, c'est potentiellement une anomalie : un numéro avec trop de mouvements.

Étape 4 : Détail des doublons

SELECT mh.msisdn, mh.date
FROM porta_msisdn_historique mh
WHERE mh.msisdn IN (
      SELECT msisdn FROM porta_msisdn_historique
      GROUP BY msisdn
      HAVING COUNT(*) > 2
  )
ORDER BY mh.msisdn, mh.date;

Note : C'est une sous-requête (un SELECT dans un autre SELECT). Le SELECT interne trouve les MSISDN suspects, puis le SELECT externe affiche les détails.

Étape 5 : Fichiers SYNC récents

SELECT f.nom, f.date_creation,
       f.expediteur, f.destinataire,
       COUNT(s.id) AS nb_msisdn
FROM porta_fichier f
LEFT JOIN porta_sync s ON s.operateur_id = f.expediteur
WHERE f.type = 'sync'
GROUP BY f.id, f.nom, f.date_creation, f.expediteur, f.destinataire
ORDER BY f.date_creation DESC
LIMIT 20;

Étape 6 : Croiser SYNC et portages

Vérifier qu'un MSISDN signalé en conflit a bien un portage correspondant :

SELECT
    s.msisdn,
    s.date AS date_sync,
    p.portage_date_souhaitee AS date_portage_reel,
    e.etat_name AS etat_portage,
    CASE
        WHEN p.id IS NULL THEN 'PAS DE PORTAGE TROUVE'
        WHEN e.etat_name = 'termine' THEN 'OK - Termine'
        ELSE 'ATTENTION - ' || e.etat_name
    END AS diagnostic
FROM porta_sync s
JOIN porta_sync_status ss ON s.sync_status = ss.id
LEFT JOIN porta_portage p ON p.portage_msisdn = s.msisdn
LEFT JOIN porta_etat e ON e.etat_name = p.portage_etat
WHERE ss.statut_name = 'LOCKED'
ORDER BY s.msisdn;
